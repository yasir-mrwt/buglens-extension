TestPilot Quick Start

TestPilot captures one active inspected page and QA session at a time.

  1. Open the target web application.
  2. Open Chrome DevTools.
  3. Select the TestPilot panel.
  4. Click Start Session.
  5. Reload the page to capture full network traffic.
  6. Perform the manual QA flow.
  7. Click Scan UI from Findings on important screens.
  8. Review Actionable findings before Needs Review and Framework Noise.
  9. Optionally use AI Chat to ask which issues are real bugs, what can be ignored, and what to test next.
  10. Stop the session, open Reports, then export HTML, JSON, or CSV.

Session data is temporary and stored locally in the browser. TestPilot does not send captured evidence to a server. UI scanning is manual to keep CPU use low.

Response body snippets are disabled by default, large or binary bodies are skipped, and sensitive headers and fields are redacted in reports.

Framework noise does not increase the main issue count by default. CSV exports are formatted for Google Sheets without requiring Google OAuth.

AI is optional. When used, TestPilot sends only a small sanitized session summary to the local backend at http://localhost:8787; raw headers, bodies, cookies, tokens, and screenshots are excluded.
